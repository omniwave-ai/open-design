# Humanize PPT public writing / propagation notes

Use when drafting public-facing articles, scripts, README copy, or launch posts about Humanize PPT.

## Core positioning

Humanize PPT is not a fixed bundle of 4 PPT Skills and not a slide renderer. It is a PPT outline/director layer that adapts to different upstream outline-producing Skills and turns their outputs into a cleaner production contract for downstream HTML PPT/rendering/video tools.

Preferred phrasing:

- `Humanize PPT能适应不同Skill交出来的大纲，把它们整理成更适合PPT生产的契约。`
- `可以推荐搭配guizang、Zara、其他HTML PPT Skill、Frontend Design、HyperFrames等，但这些是建议搭配，不是产品边界。`
- `它像导演/编辑，负责删掉模型草稿纸、保留判断、把功能翻译成场景、把大纲变成观众听得懂的叙事。`

Avoid:

- `Humanize PPT只支持/绑定这4个Skill。`
- `Humanize PPT就是一个PPT生成器。`
- `把推荐Skill清单写成产品定义。`

## Article angle that worked

Two-article split:

1. **WorkBuddy / Agent Teams / OPC angle**
   - Open with a concrete scene such as `几个AI朋友被腾讯录用了`.
   - Use WorkBuddy专家团 to explain one-person-company as `一个人调度AI团队`, not `一个人熬命做所有活`.
   - Mention the real chain only as proof: outline → HTML PPT → speaker notes → screenshots → HyperFrames video → install video script/TTS.
   - End on: `一人公司的OPC确实是一个越来越简单的事，每个人都可以做。`

2. **Humanize PPT product angle**
   - Open with the concrete pain: GPT 5.5/other models put useless thinking process or draft notes into PPT pages.
   - Explain Humanize PPT as the middle layer between many Skills and renderers.
   - Use a real deck case to show how it turns features into scenes: Agent Browser = 网页不是文本，是现场; Office docs = 真正耗时的是文件格式; Web Search = 最近/最新/现在就先搜索再回答.
   - Emphasize that AI-generated content often has too much content but not enough human judgment.

## Style reminders for Carl-facing drafts

- Keep the narrative concrete and first-person. Do not turn Skill releases into feature-list posts.
- Do not overuse mechanical sections like `case一、case二、case三`.
- For Chinese prose, avoid mechanical spaces around product/model names unless part of the name itself.
- Put recommended pairings after the positioning is clear, so readers do not mistake examples for limits.
